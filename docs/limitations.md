# FlowForge n8n - Limitations

This document lists the known limitations of FlowForge n8n.

---

## 1. Scaffolding & Mappings
*   **Version Drift:** n8n updates node configurations and versions frequently. While FlowForge outputs schemas matching standard v1+ formats, some specific target versions inside self-hosted nodes may require manual edits inside the visual UI.
*   **Expression Scopes:** FlowForge does not evaluate dynamic JavaScript expressions during offline lint check runs (e.g., `{{ $json.myKey }}`). Expressions are validated strictly on syntax, not actual run-time values.

---

## 2. Webhook Testing
*   **Local Webhooks Execution:** Triggering webhook URLs requires an active n8n instance running locally or on a reachable staging URL. Curl scripts generated by the tool cannot execute successfully if n8n is offline.
*   **Test URLs:** Test URLs inside n8n only remain open for a short time after you click the **"Listen for test event"** button. If you execute testing scripts before or after this window, you will receive a `404 Webhook not found` error.

---

## 3. Custom Nodes Scaffolder
*   **Boilerplate Only:** `flowforge node-new` generates clean TypeScript structures and package files. However, you must implement the specific request logic inside the node executor functions yourself.
